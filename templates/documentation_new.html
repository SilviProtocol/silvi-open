Overview

Treekipedia GraphFlow transforms biodiversity data from multiple sources into semantically rich OWL ontologies and manages them in a graph database.

Multi-Source Integration
  • Google Sheets (collaborative)
  • PostgreSQL (enterprise scale)
  • CSV Files (quick imports)
Smart Analysis
  • 120+ field patterns
  • 8 ontology classes
  • Automatic categorization
Format Flexibility
  • Direct schema (columns)
  • Transposed (rows)
  • MVP metadata format
Triple Store Ready
  • Apache Jena Fuseki
  • SPARQL endpoints
  • Incremental updates
Recent Improvements (v2.0)
  • NEW Automatic transposed format detection
  • FIXED Accurate row/column counts
  • FIXED "Page not found" alert
  • IMPROVED Direct schema processing

Quick Start

1
Install Dependencies

Set up Python environment

# Clone and setup git clone <your-repo> cd biodiversity-ontology-automation # Create virtual environment python -m venv venv source venv/bin/activate # Mac/Linux # Install packages pip install -r requirements.txt
2
Configure Services

Set up Google Sheets & Fuseki

# config.py USE_GOOGLE_SHEETS = True GOOGLE_SHEETS_CREDS_FILE = 'service_account.json' FUSEKI_BASE_URL = 'http://167.172.143.162:3030' FUSEKI_DATASET = 'treekipedia'
3
Run Application

Start the Flask server

python app.py # Access at http://localhost:5001
4
Import Your Data

Generate your first ontology

Pro Tip: Your TreeSheet with transposed species data (120 rows × 1 column) will be automatically detected and processed!

API Reference

GET /api/system-status

Check status of all integrations (Fuseki, Google Sheets, PostgreSQL)

POST /import-from-sheets

Import and generate ontology from Google Sheets

Form Data: spreadsheet_id/name, ontology_name, force_generation

GET /spreadsheet-metadata

Get sheet metadata (actual row/col counts, version info)

Query: ?spreadsheet_name=TreeSheet

POST /preview-multi-sheet-ontology

Preview ontology structure before generation

Frequently Asked Questions

Q: How many fields can the system handle?

A: Tested with 120+ fields. Performance limit is approximately 500 fields for dynamic analysis.

Q: Do I need Google Sheets?

A: No! CSV upload works standalone. Google Sheets adds collaboration and change detection features.

Q: Can I customize ontology classes?

A: Yes! Edit multi_sheet_biodiversity_generator.py to add custom classes and field patterns.

Q: How do I query the generated ontology?

A: Use the SPARQL endpoint at http://167.172.143.162:3030/treekipedia/sparql

Q: What about my 50,000+ species data?

A: For actual species data rows, use PostgreSQL import. The current system processes schema definitions (field names), not data rows.
